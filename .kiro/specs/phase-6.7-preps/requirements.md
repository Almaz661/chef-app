# Requirements — Фаза 6.7: Заготовки

## Контекст

Заготовка — это полуфабрикат, который пользователь готовит **сам по
рецепту**, а потом хранит в холодильнике/морозилке и расходует по другим
рецептам как обычный продукт. Примеры: куриный бульон, песочное тесто,
пельменный фарш, зажарка, соус болоньезе, ягодное пюре.

Сценарий из жизни:
- В воскресенье пользователь готовит «Куриный бульон» (рецепт-заготовка
  в группе **PREP**).
- Бекенд списывает ингредиенты бульона из инвентаря (как любой обычный
  «cook»), но дополнительно создаёт **новую партию** продукта-заготовки
  «Куриный бульон» (1.5 л, FREEZER, expires = today + 90 дней).
- Через неделю пользователь готовит «Куриный суп» — рецепт берёт
  бульон из инвентаря, FEFO работает как обычно.

## User stories

**US-1.** *Как повар*, я хочу видеть в навигации меню восьмую кнопку
«Заготовки», чтобы быстро находить рецепты-полуфабрикаты.

**US-2.** *Как повар*, я хочу пометить продукт как заготовку
(`isPrep = true`), чтобы фронт мог отличать «куплено в магазине» от
«сделано дома».

**US-3.** *Как повар*, я хочу привязать рецепт-заготовку к
продукту-заготовке (через `producesProductId`), чтобы система знала,
что появляется в инвентаре после готовки.

**US-4.** *Как повар*, я хочу указать, **сколько** заготовки получается
из одной порции рецепта (`prepYieldQuantity` + `prepYieldUnitId`),
чтобы готовка двойной порции бульона давала вдвое больше литров.

**US-5.** *Как повар*, я хочу указать **где** хранится заготовка по
умолчанию (`prepDefaultLocation`: FRIDGE / FREEZER), чтобы система
сразу клала партию в правильное место.

**US-6.** *Как повар*, я хочу указать **сколько дней** заготовка живёт
(`prepShelfLifeDays`), чтобы автоматически проставлялся `expiresAt` и
работали FEFO + alerts.

**US-7.** *Как повар*, я хочу, чтобы по нажатию `cook` для
PREP-рецепта в одной транзакции:
- ингредиенты списались из инвентаря (как обычно),
- создалась партия продукта-заготовки с `expiresAt` и нужной локацией,
- появился `InventoryTxn` про эту партию (для истории).

**US-8.** *Как повар*, я хочу, чтобы для рецептов **без**
`producesProductId` (обычные блюда) поведение `cook` не менялось ни на
байт — обратная совместимость с фазами 3–6.6.

## Стандартные сроки годности (значения по умолчанию для seed)

| Категория заготовки                 | Дни  |
|-------------------------------------|------|
| Мясо / рыба / бульоны / фарш        | 90   |
| Тесто (сырое)                       | 30   |
| Овощные базы / соусы / пюре         | 180  |

Это **значения для seed**, не «жёстко зашитая» константа в коде. Поле
`prepShelfLifeDays` хранится на каждом рецепте отдельно, шеф может
выставить любое число.

## Acceptance Criteria

- [ ] enum `RecipeGroup` содержит 8 значений: 7 старых + `PREP`.
- [ ] на `Product` есть `isPrep: boolean` (default `false`); существующие
  100+ продуктов остаются с `isPrep = false`.
- [ ] на `Recipe` есть набор полей:
      - `producesProductId: String?` (FK на `Product`),
      - `prepYieldQuantity: Decimal?`,
      - `prepYieldUnitId: String?` (FK на `Unit`),
      - `prepDefaultLocation: InventoryLocation?`,
      - `prepShelfLifeDays: Int?`.
      Поля **либо все заполнены** (PREP-рецепт), либо все NULL (обычный
      рецепт). Это инвариант — проверяется в сервисе и в integrity-тестах.
- [ ] миграция накатывается чисто (нет `ALTER TYPE` без значения).
- [ ] `CookingService.cook(menuRecipeId)` для PREP-рецепта в той же
  транзакции, что и списание ингредиентов:
      - создаёт `InventoryItem`: `productId = recipe.producesProductId`,
        `quantity = prepYieldQuantity * scale`, `unitId = prepYieldUnitId`,
        `location = prepDefaultLocation`,
        `expiresAt = now + prepShelfLifeDays`,
        `acquiredAt = now`;
      - пишет `InventoryTxn` с положительным `quantity`,
        `source = PREP_PRODUCTION`, `refType = 'MenuRecipe'`,
        `refId = menuRecipeId`.
      Ответ `cook()` дополнительно содержит поле `produced: { ... }`.
- [ ] для рецептов без `producesProductId` ответ `cook()` остаётся
  неизменным (`produced` отсутствует / `null`).
- [ ] в seed добавлены 10 продуктов-заготовок (`isPrep = true`) и
  10 рецептов-заготовок (`group = PREP`, `producesProductId` указывает на
  соответствующий продукт-заготовку).
- [ ] сроки годности в seed соответствуют таблице выше.
- [ ] тесты:
      - `prep-yield.spec.ts` — чистая функция `computePrepYield`
        (масштабирование, локация, дата истечения);
      - расширение `catalog-integrity.spec.ts` — каждый PREP-рецепт
        корректно ссылается на продукт с `isPrep = true`, поля
        `prepYield*` / `prepDefaultLocation` / `prepShelfLifeDays`
        заполнены;
      - регрессия: все ранее зелёные тесты остаются зелёными.
- [ ] PR #9 открыт.

## Out of scope

- Auto-перевод заготовок из FREEZER в FRIDGE при разморозке.
- «Половинчатая» заготовка (распилить партию пополам перед готовкой).
- Расчёт KBJU заготовки из её ингредиентов (используем поля заготовки
  как обычного продукта — KBJU выставляются вручную в seed).
- UI — это бекенд-фаза.
