# Фаза 1 — Привязка рецептов к Product Master

## 1. Контекст и цель

В ШефДоме рецепты хранятся как набор «свободных» строк ингредиентов
(например, `"2 ст.л. оливкового масла"`). Это блокирует:

- расчёт КБЖУ и стоимости порции,
- агрегированный shopping-list по нескольким рецептам,
- замену продуктов по аллергенам/диете,
- аналитику «что чаще всего готовится из X».

**Цель Фазы 1:** ввести **Product Master** (единый справочник продуктов) и
заменить «строковые» ингредиенты в рецептах на ссылки на конкретный
`product_id`, плюс структурированные `quantity` + `unit`. Старая
человекочитаемая строка остаётся как `raw_text` (для отображения и аудита).

## 2. Глоссарий

| Термин | Значение |
| --- | --- |
| Product | Канонический продукт в справочнике (например, «Масло оливковое Extra Virgin»). |
| ProductAlias | Альтернативное написание/синоним продукта (например, «оливковое масло», «olive oil»). Используется для автомэтча. |
| Unit | Единица измерения (`g`, `kg`, `ml`, `l`, `tbsp`, `tsp`, `pcs`, …). |
| UnitConversion | Правило перевода между единицами в контексте конкретного продукта (например, для «Мука»: 1 ст.л. = 25 г). |
| Recipe | Рецепт. |
| RecipeIngredient | Позиция в рецепте: `(recipe, product, quantity, unit, raw_text, note)`. |
| IngredientMatchQueue | Очередь строк ингредиентов, которые автомэтч не смог однозначно привязать к Product. Разбирается вручную. |

## 3. Требования

### 3.1 Функциональные

- **FR-1.** Существует справочник `Product` с уникальным `slug`, названием,
  категорией, базовой единицей (`base_unit`), опциональными КБЖУ на 100 г
  и тегами (`vegan`, `gluten_free`, …).
- **FR-2.** Существует `ProductAlias` — many-to-one к `Product`, уникальный
  `(normalized_text, locale)`. Минимум один алиас на продукт — само его имя.
- **FR-3.** Существует справочник `Unit` (`g`, `kg`, `ml`, `l`, `tbsp`,
  `tsp`, `pcs`, `pinch`) и `UnitConversion` для перевода между ними:
  - глобальные правила (`1 kg = 1000 g`, `1 l = 1000 ml`),
  - продуктово-специфичные правила (`Мука: 1 tbsp = 25 g`).
- **FR-4.** `RecipeIngredient` обязан ссылаться на `Product` (FK
  `product_id NOT NULL`) и хранить `quantity` (decimal, > 0) и `unit_id`.
  Поле `raw_text` хранит исходную человекочитаемую строку.
- **FR-5.** Сервис парсинга строки ингредиента превращает
  `"2 ст.л. оливкового масла"` в структурированный объект
  `{ quantity: 2, unit: 'tbsp', name: 'оливковое масло', note?: string }`.
- **FR-6.** Сервис матчинга по входному `name` пытается:
  1. точный матч `ProductAlias.normalized_text`,
  2. fuzzy-матч (trigram, порог настраиваемый, по умолчанию `>= 0.45`),
  3. если ничего — кладёт запись в `IngredientMatchQueue`.
- **FR-7.** REST API:
  - `GET/POST/PATCH/DELETE /products`
  - `GET/POST/DELETE /products/:id/aliases`
  - `GET/POST/PATCH/DELETE /recipes`
  - `POST /recipes/:id/ingredients` — принимает массив строк и/или
    структурированных позиций; неразрешённые попадают в очередь.
  - `GET /matching/queue` — список нерешённых элементов.
  - `POST /matching/queue/:id/resolve` — оператор привязывает строку к
    `product_id` (опционально создавая алиас).
- **FR-8.** Seed создаёт базовые `Unit` и пару демо-продуктов (соль, мука,
  оливковое масло, помидор) с алиасами, плюс один пример рецепта.

### 3.2 Нефункциональные

- **NFR-1.** TypeScript strict, NestJS 10.x, Node.js 20.
- **NFR-2.** PostgreSQL 15+, расширение `pg_trgm` обязательно (для fuzzy).
- **NFR-3.** Все миграции — Prisma migrations, никакого ручного DDL «мимо».
- **NFR-4.** Локаль данных по умолчанию `ru`; модель готова к мультилокали
  через `ProductAlias.locale`.
- **NFR-5.** Вся бизнес-логика покрыта unit-тестами в последующих фазах
  (в Фазе 1 тесты не обязательны; модули спроектированы тестируемыми).

## 4. Из области не входят (Out of scope для Фазы 1)

- Frontend / UI.
- Аутентификация и мультипользовательность.
- Импорт рецептов из внешних источников (сайтов).
- Расчёт КБЖУ всего рецепта (логика будет в Фазе 2).
- Shopping-list (Фаза 3).

## 5. Критерии приёмки

- [ ] `prisma migrate dev` поднимает чистую схему без ошибок.
- [ ] `npm run seed` наполняет БД базовыми единицами, продуктами,
      алиасами и одним примером рецепта; все ингредиенты примера
      смэтчены (никто не упал в `IngredientMatchQueue`).
- [ ] Создание рецепта со строкой `"2 ст.л. оливкового масла"` через
      `POST /recipes/:id/ingredients` приводит к корректной
      `RecipeIngredient` с `product_id` оливкового масла,
      `quantity=2`, `unit='tbsp'`.
- [ ] Создание рецепта со строкой `"1 пачка непонятного"` создаёт запись
      в `IngredientMatchQueue` и не ломает запрос.
- [ ] `POST /matching/queue/:id/resolve` корректно создаёт
      `RecipeIngredient` и (опционально) `ProductAlias`.
