# Фаза 6.6 — Группы рецептов для навигации

## 1. Цель

Дать фронтенду **семь кнопок навигации** по типу блюда, чтобы пользователь
не листал общую кашу из всех рецептов:

- Завтраки
- Супы
- Основные блюда
- Салаты
- Выпечка
- Десерты
- Напитки

Плюс неявная кнопка «Все», которая просто не фильтрует.

## 2. Функциональные требования

- **FR-6.6.1.** У `Recipe` появляется поле `group: RecipeGroup?` со
  значениями из закрытого перечня (см. ниже). Поле опциональное —
  существующие рецепты без него остаются валидными (отображаются как
  `UNGROUPED` в счётчиках; в фильтрованной выдаче не появляются).
- **FR-6.6.2.** `GET /recipes` принимает query-параметры:
  - `group` — точное совпадение по перечню. Без значения = «Все».
  - `search` — подстрока в `title` или `description`,
    регистронезависимо. Пробельные строки игнорируются.
  - Параметры объединяются через AND.
- **FR-6.6.3.** `GET /recipes/groups` возвращает массив счётчиков
  `[{ group, count }]`, по одной строке на каждую известную группу +
  `UNGROUPED`. Порядок фиксированный — фронт может отрисовать кнопки
  как есть, без сортировки.
- **FR-6.6.4.** `POST /recipes` и `PATCH /recipes/:id` принимают
  опциональное поле `group` в теле — для ручного редактирования из
  фронтенда.
- **FR-6.6.5.** Демо-данные seed: 5 уже существующих рецептов получают
  `group`, плюс добавляется минимум 10 новых рецептов так, чтобы каждая
  из 7 групп содержала **минимум 2** блюда. Сейчас фактически 19
  рецептов: BREAKFAST 3 / SOUP 3 / MAIN 4 / SALAD 3 / BAKING 2 /
  DESSERT 2 / DRINK 2.

## 3. Перечень групп

```
enum RecipeGroup {
  BREAKFAST   // Завтраки
  SOUP        // Супы
  MAIN        // Основные блюда
  SALAD       // Салаты
  BAKING      // Выпечка
  DESSERT     // Десерты
  DRINK       // Напитки
}
```

Перечень закрытый. Расширение перечня — отдельная фаза с миграцией.
Подкатегории внутри группы (например, «горячий суп» vs «холодный
суп») в эту фазу не входят.

## 4. Нефункциональные

- **NFR-6.6.1.** Один индекс на колонку `group` для быстрых выборок.
- **NFR-6.6.2.** Никакого N+1 в `groupCounts` — один `groupBy`.

## 5. Out of scope

- Кухня (русская / итальянская / …).
- Сложность и время приготовления.
- Фильтр по диетическим тегам в той же ручке (есть `diet-check`).
- Полнотекстовый поиск по ингредиентам.
- Пагинация списка рецептов (если их будет много — отдельная фаза).

## 6. Критерии приёмки

- [ ] Миграция `20260519000000_recipe_groups` поднимает enum и колонку.
- [ ] `GET /recipes?group=BREAKFAST` возвращает только завтраки.
- [ ] `GET /recipes?search=суп` находит «Чечевичный суп» и «Куриный
      суп с лапшой» по подстроке в названии.
- [ ] `GET /recipes/groups` возвращает 8 строк (7 групп + UNGROUPED) в
      фиксированном порядке, с корректными счётчиками.
- [ ] Все catalog-integrity тесты зелёные, включая новые: «у каждого
      рецепта валидная группа», «каждая группа имеет ≥1 рецепт».
- [ ] Регрессия: 85 предыдущих тестов остаются зелёными.
