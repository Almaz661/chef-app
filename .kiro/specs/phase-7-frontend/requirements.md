# Requirements — Фаза 7: Минимальный фронтенд

## Контекст

9 фаз бэкенда дали полноценный REST API, но пользователь не может
«потыкать» систему без curl/Postman. Фаза 7 — это минимальный
кликабельный UI, который визуализирует основные возможности:
навигацию по рецептам, карточку с КБЖУ, инвентарь, алерты сроков
и историю готовки.

Фронтенд живёт в `web/` в том же монорепо. Общается с NestJS бэкендом
через HTTP (proxy через Next.js rewrites, без CORS).

## User stories

**US-1.** *Как пользователь*, я вижу 8 кнопок-групп рецептов (Завтраки,
Супы, Основные, Салаты, Выпечка, Десерты, Напитки, Заготовки) с
количеством рецептов в каждой.

**US-2.** *Как пользователь*, я могу кликнуть на группу и увидеть
список рецептов с поиском по названию.

**US-3.** *Как пользователь*, я открываю карточку рецепта и вижу:
- список ингредиентов с количеством и единицами;
- КБЖУ на порцию (ккал / белки / жиры / углеводы);
- для заготовок — что производится, сколько, где хранится, срок.

**US-4.** *Как пользователь*, я вижу страницу инвентаря, разбитую по
локациям (холодильник, морозилка, кладовка) с количеством и сроками.

**US-5.** *Как пользователь*, я вижу страницу алертов — продукты,
срок которых истекает через N дней (настраиваемо: 1/3/7/14).

**US-6.** *Как пользователь*, я вижу историю готовки за последний
месяц: что готовил, когда, сколько порций, что израсходовано.
Плюс сводка: топ-5 рецептов.

## Acceptance Criteria

- [ ] `npm run dev` в `web/` поднимает Next.js на :3001 (или default :3000).
- [ ] Все 6 страниц рендерятся без JS-ошибок (при доступном API).
- [ ] При недоступном API — graceful fallback (пустое состояние, не crash).
- [ ] Навигация между страницами — без полной перезагрузки (client-side).
- [ ] Tailwind-стили минимальны но читаемы (mobile-first responsive).
- [ ] Monorepo: бэкенд в корне, фронт в `web/`, отдельные package.json.

## Out of scope

- Auth / login.
- Создание/редактирование рецептов через UI.
- Добавление в меню / shopping list через UI.
- Кнопка «Приготовить» с полным флоу (требует создание MenuRecipe).
- PWA / service worker.
- i18n (только русский).
- SSR с реальной БД (все данные через client-side fetch).
